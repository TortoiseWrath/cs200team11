Glossary

  • Provider : A health care professional providing services to its members
  • Terminal : The access point for each provider to the system each terminal is equipped with a card reader
  • Fee : can be either a subscription fee - A monthly sum payment for services / or a service fee - one time payment for services
  • Member Number : 9 digit number used to identify each member
  • Provider Number : 9 digit number used to identify each provider
  • Member Card : Encoded magnetic strip that stores all members data
  • ChocAn Data Center : central computer dialed by the terminal to find member information
  • Member Status : Can be one of two values - suspended or un-suspended
  • Provider directory : used to look up the appropriate 6 digit service code
  • Service Code : 6 digit number used to identify services with names up to 20 characters
  • Service : The services provided by providers that is identified by a services code and a name of up to 20 characters
  • Record : EFT data written to disk during the main accounting procedure
  • Electronic Funds Transfer (EFT) : data that is written to a disk; banking computers will later ensure that each providers bank account is credited with the appropriate amount
  • Main Accounting procedure : A report generated by the ChocAn data center at midnight every Friday
  • Member Report : List of services provided to each member that is sent to each member during the main accounting procedure
  • Provider Report : A list of services provided by each provider during the main accounting procedure
  • Summary Report : Given to the manager for accounts payable. A list of every provider to be paid that week, the number of consultations each provider had and his or her total fee for that week
  • Interactive Mode : The state of the software at the ChocAn data center during the day that allows operators to update member and provider records.
  • Operator : actors at the ChocAn data center interacting with the software in interactive mode
  • Acme Accounting Services : A third party organization responsible for financial procedures
  • Membership Records : Records stored at the ChocAn data center that can be updated by operators in interactive mode and are updated at 9 PM each evening by ACME accounting services

Use Cases

 
		

		
Use case: Member card swiped to receive services
Context: To receive services a member hands his or her card to the provider, who slides the card through the card reader which validates the members status.
Actors: Member, provider, and ChocAn
Main Success Scenario:
1.	Member hands card to service provider
2.	Provider slides card through card reader on terminal
	2.1.		Terminal dials ChocAn data center
	2.2.		ChocAn data center verifies member number
3.	"Validated" appears on display
		
Extensions:
2.2a.	Number is not valid
	2.2a.1.	"Invalid Number" displayed
	2.2a.2.	"Member Suspended" displayed - Indicates fees are owed and member status set to "suspended"


Use case: Bill ChocAn for health care services provided to member
Context: To bill ChocAn after a health care service has been provided to the member, the provider again passes the card through the card reader or keys in the member number. When the word Validated appears, the provider keys in the date the service was provided
Actors: Member, provider, and ChocAn
Main Success Scenario:
1.	Member hands card to service provider
2.	Provider passes card through card reader
	2.2a. Provider keys in member number
3.	Word "Validated" appears
4.	Provider keys in date service was provided in format MM-DD-YYYY
5.	Provider uses the Provider Directory to look up the appropriate six-digit service code corresponding to the service provided
6.	Provider keys in service code
7.	Software product then displays the name of the service corresponding to the code (up to 20 characters)
	7.1.		Software product asks the provider to verify that this is indeed the service that was provided
8.	Provider has option to enter comments about service provided
9.	Software product writes a record to disk that includes the following fields:
	Current date and time (MM–DD–YYYY HH:MM:SS).
	Date service was provided (MM–DD–YYYY).
	Provider number (9 digits).
	Member number (9 digits).
	Service code (6 digits).
	Comments (100 characters) (optional)
10.	Software product next looks up the fee to be paid for that service and displays it on the provider’s terminal
11.	For verification purposes, the provider has a form on which to enter the current date and time, the date the service was provided, member name and number, service code, and fee to be paid
12.	 At the end of the week, the provider totals the fees to verify the amount to be paid to that provider by ChocAn for that week
		
Extensions: Possible alternatives to the steps you may think of (e.g., what happens if something goes wrong at this step), such as:
7.1a.	If the provider has entered a nonexistent code, an error message is printed
