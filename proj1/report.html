Glossary

Acme Accounting Services
A third party organization responsible for financial procedures
ChocAn Data Center
Central computer dialed by the terminal to find member information
Electronic Funds Transfer (EFT)
Data that is written to a disk; banking computers will later ensure that each providers bank account is credited with the appropriate amount
Fee
Can be either a subscription fee - A monthly sum payment for services / or a service fee - one time payment for services
Interactive Mode
The state of the software at the ChocAn data center during the day that allows operators to update member and provider records.
Main Accounting procedure
A report generated by the ChocAn data center at midnight every Friday
Member Card
Encoded magnetic strip that stores all members data
Member Number
9 digit number used to identify each member
Member Report
List of services provided to each member that is sent to each member during the main accounting procedure
Member Status
Can be one of two values - suspended or un-suspended
Membership Records
Records stored at the ChocAn data center that can be updated by operators in interactive mode and are updated at 9 PM each evening by ACME accounting services
Operator
Actors at the ChocAn data center interacting with the software in interactive mode
Provider directory
Used to look up the appropriate 6 digit service code
Provider Number
9 digit number used to identify each provider
Provider Report
A list of services provided by each provider during the main accounting procedure
Provider
A health care professional providing services to its members
Record
EFT data written to disk during the main accounting procedure
Service Code
6 digit number used to identify services with names up to 20 characters
Service
The services provided by providers that is identified by a services code and a name of up to 20 characters
Summary Report
Given to the manager for accounts payable. A list of every provider to be paid that week, the number of consultations each provider had and his or her total fee for that week
Terminal
The access point for each provider to the system each terminal is equipped with a card reader

Use cases

Member card swiped to receive services

To receive services a member hands his or her card to the provider, who slides the card through the card reader which validates the members status.

Member, provider, and ChocAn Server

  1. Member hands card to service provider
  2. Provider slides card through card reader on terminal
    1. Terminal dials ChocAn data center
    2. ChocAn data center verifies member number
  3. Validated appears on display
  1. Number is not valid
    1. Invalid Number displayed
    2. Member Suspended displayed - Indicates fees are owed and member status set to "suspended"

Bill ChocAn for health care services provided to member

To bill ChocAn after a health care service has been provided to the member, the provider again passes the card through the card reader or keys in the member number. When the word Validated appears, the provider keys in the date the service was provided.

Member, provider, and ChocAn Server

  1. Member hands card to service provider
  2. Provider slides card through card reader on terminal or keys in member number
  3. Validated appears on display
  4. Provider keys in date service was provided in format MM-DD-YYYY
  5. Provider uses the Provider Directory to look up the appropriate six-digit service code corresponsing to the service provided
  6. Provider keys in the service code
  7. Software product then displays the name of the service corresponding to the code (up to 20 characters)
    1. Software product asks the provider to verify that this is indeed the service that was provided
  8. Provider enters comments about the service provided (optional)
  9. Software product writes a record to disk that includes the following fields:
    • Current date and time (MM-DD-YYYY HH:MM:SS)
    • Date service was provided (MM-DD-YYYY)
    • Provider number (9 digits)
    • Member number (9 digits)
    • Service code (6 digits)
    • Comments (up to 100 characters)
  10. Software product looks up the fee to be paid for that service and displays it on the provider's terminal
  11. For verification purposes, the provider has a form on which to enter the current date and time, the date the service was provided, member name and number, service code, and fee to be paid
  12. At the end of the week, the provider totals the fees to verify the amount to be paid to that provider by ChocAn for that week
  1. If the provider has entered a nonexistent code, an error message is printed

Provider Directory Request

Provider makes request to software for the provider directory, an alphabetically ordered list of service names and corresponding service codes and fees. The Provider Directory is sent to the provider as an e-mail attachment.

Provider and ChocAn Server

  1. Provider uses terminal to request provider directory
  2. Software sends provider directory to provider as e-mail attachment

Main accounting procedure ran

At midnight on Friday, the main accounting procedure is run at the ChocAn Data Center. It reads the week’s file of services provided and prints a number of reports. Each report also can be run individually at the request of a ChocAn manager at any time during the week.

Member, Provider, and ChocAn Server

  1. Main accounting procedure ran on midnight of Friday
    1. ChocAn manager can request report at any time
  2. Read the weeks file of services provided
  3. Prints a number of reports
  4. Record consisting of EFT data written to disk
    1. Banking computers later ensure that each providers bank account is credited with the proper amount

Member report sent

List of services provided sent to member from ChocAn

Member and ChocAn Server

  1. Report sent to each member that received services in last week
  2. Report also sent as e-mail attachment
    • Member name (25 characters)
    • Member number (9 digits)
    • Member street address (25 characters)
    • Member city (14 characters)
    • Member state (2 letters)
    • Member ZIP code (5 digits)
    • For each service provided, the following details are required:
    • Date of service (MM–DD–YYYY)
    • Provider name (25 characters)
    • Service name (20 characters)

Provider report sent

List of services provided to members sent to each provider

Provider and ChocAn Server

  1. Report sent to each provider that has billed ChocAn in last week
    • Provider name (25 characters)
    • Provider number (9 digits)
    • Provider street address (25 characters)
    • Provider city (14 characters)
    • Provider state (2 letters)
    • Provider ZIP code (5 digits)
    • For each service provided, the following details are required:
    • Date of service (MM–DD–YYYY)
    • Date and time data were received by the computer (MM–DD–YYYY HH:MM:SS)
    • Member name (25 characters)
    • Member number (9 digits)
    • Service code (6 digits)
    • Fee to be paid (up to $999.99)
    • Total number of consultations with members (3 digits)
    • Total fee for week (up to $99,999.99)
  2. Summary report is given to manager for accounts payable
    • lists every provider to be paid that week, the number of consultations each had, and his or her total fee for that week
  3. Total number of providers who provided services, the total number of consultations, and the overall fee total printed

Member/Provider records are added, deleted, or updated

During the day, the software at the ChocAn Data Center is run in interactive mode to allow operators to add, delete, or update member of provider records

Operator and ChocAn Server

  1. ChocAn data center ran in interactive mode
  2. Record added, deleted, or updated
    • Member
    • Provider

ACME updates member records

The processing of payments of ChocAn membership fees has been contracted out to Acme. The Acme computer updates the relevant ChocAn Data Center computer membership records each evening at 9 P.M

Member, Acme, and ChocAn Server

  1. Acme records payment of membership fees
  2. Acme suspends users whose fees are overdue
  3. Acme reinstates suspended members who have paid what is owing
  4. Acme computer updates the relevant ChocAn Data Center computer membership records each evening at 9 P.M.
